# 🔍 Поисковый движок с булевыми запросами

Полнофункциональный поисковый движок на Python с поддержкой обратного и координатного индексов, булевых запросов и веб-интерфейсом.

## ✨ Основные возможности

### 🏗️ Архитектура поисковой системы
- **Обратный индекс** (Inverted Index) - для быстрого поиска документов по терминам
- **Координатный индекс** (Positional Index) - для поиска с ограничением расстояния между словами
- **TF-IDF ранжирование** - для сортировки результатов по релевантности
- **Поддержка полей документа** - поиск по метаданным (автор, категория, заголовок)

### 🔎 Типы запросов
- **Простой поиск**: `Python`
- **Логическое И**: `Python AND programming`
- **Логическое ИЛИ**: `Python OR Java`
- **Исключение**: `Python NOT web`
- **Поиск по полю**: `category:technology`, `author:Smith`
- **Комбинированный**: `author:Expert title:Machine category:technology`
- **С ограничением расстояния**: `Python programming` (максимальное расстояние между словами)

### 🌐 Веб-интерфейс
- Современный responsive дизайн
- Мгновенный поиск при вводе
- Подсветка результатов с метаданными
- API для интеграции с другими системами

## 🏛️ Архитектура системы

### Компоненты поискового движка

#### 1. DocumentIndexer
Отвечает за создание и управление индексами:
- Токенизация и нормализация текста
- Построение обратного индекса (термы → документы)
- Построение координатного индекса (термы → позиции в документах)
- Вычисление TF-IDF метрик

#### 2. BooleanQueryParser
Парсит булевы запросы:
- Поддержка операторов AND, OR, NOT
- Обработка полей документов (field:value)
- Токенизация и нормализация запросов

#### 3. SearchEngine
Основной класс поисковой системы:
- Выполнение булевых запросов
- Поиск с ограничением расстояния между термами
- Ранжирование результатов по TF-IDF

### Алгоритмы и структуры данных

#### Обратный индекс
```python
{
  "python": {"doc1", "doc2", "doc5"},
  "machine": {"doc1", "doc3"},
  "learning": {"doc1", "doc3", "doc7"}
}
```

#### Координатный индекс
```python
{
  "python": {
    "doc1": [0, 15, 32],  # позиции слова в документе
    "doc2": [5, 21]
  }
}
```

## 🚀 Использование

### Примеры API запросов

```python
# POST /search
{
  "query": "machine learning AND category:technology",
  "max_distance": 5  # опционально
}

# Ответ
{
  "query": "machine learning AND category:technology",
  "total_results": 2,
  "results": [
    {
      "id": "doc1",
      "title": "Introduction to Machine Learning",
      "author": "Tech Expert", 
      "category": "technology",
      "score": 2.847,
      "snippet": "Machine learning algorithms..."
    }
  ]
}
```

### Добавление новых документов


## 🔧 Конфигурация и настройка

### Параметры поиска
- `max_distance`: максимальное расстояние между терминами (по умолчанию: без ограничений)
- TF-IDF параметры настраиваются в классе `DocumentIndexer`

### Расширение функциональности
Система спроектирована для легкого расширения:

1. **Поддержка опечаток**: добавить edit distance в `BooleanQueryParser`
2. **Джокерные символы**: расширить парсер запросов для `*` и `?`
3. **Фразовые запросы**: добавить поддержку кавычек `"exact phrase"`
4. **Автодополнение**: создать индекс префиксов терминов
5. **Синонимы**: интегрировать словарь синонимов
