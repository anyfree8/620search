# 🔎 620search

Минимальный поисковый движок на Python с булевой логикой и оператором NEAR. Веб-интерфейс на Flask, данные читаются из protobuf-индексов Wikipedia (обратный и позиционный индексы).

## ✨ Основные возможности

### 🏗️ Архитектура
- `dev/index.py`
  - `ReverseIndex` — обратный индекс из protobuf; свойства: `total_terms`, `index_size`
  - `CoordinateIndex` — позиционный индекс из protobuf
  - `DataIndex` — коллекция документов: `get(doc_id) -> {doc_id, title, text, url}`, `total_documents`
- `dev/search_engine.py` — парсинг и исполнение булевых/NEAR запросов поверх индексов
- `templates/` — обновлённые интерфейсы (`index.html`, `document.html`) с тёмной темой
- `app.py` — основной Flask-приложение

### 🔎 Синтаксис запросов
- Простой: `term`
- Булевы: `term1 AND term2`, `term1 OR term2`, `term1 AND NOT term3`
- Поля: `title::литва`, `text::москва`
- NEAR (окно k): `term1 term2 @k` (пример: `title::Партия труда @2`)

### 🌐 Веб-интерфейс
- Современный UI с тёмной/светлой темой (переключатель 🌓)
- Аккордеон «Синтаксис запросов», быстрые чипы-примеры
- Результаты: `title`, `url`, `snippet`, `score`

## 🏛️ Архитектура системы

### Компоненты поискового движка

#### 1. DocumentIndexer
Отвечает за создание и управление индексами:
- Токенизация и нормализация текста
- Построение обратного индекса (термы → документы)
- Построение координатного индекса (термы → позиции в документах)
- Вычисление TF-IDF метрик

#### 2. BooleanQueryParser
Парсит булевы запросы:
- Поддержка операторов AND, OR, NOT
- Обработка полей документов (field:value)
- Токенизация и нормализация запросов

#### 3. SearchEngine
Основной класс поисковой системы:
- Выполнение булевых запросов
- Поиск с ограничением расстояния между термами
- Ранжирование результатов по TF-IDF

### Алгоритмы и структуры данных

#### Обратный индекс
```python
{
  "python": {"doc1", "doc2", "doc5"},
  "machine": {"doc1", "doc3"},
  "learning": {"doc1", "doc3", "doc7"}
}
```

#### Координатный индекс
```python
{
  "python": {
    "doc1": [0, 15, 32],  # позиции слова в документе
    "doc2": [5, 21]
  }
}
```

## TF-IDF ranking

Для первого рабочего варианта используется устойчивая сглаженная формула TF-IDF:

$$
\mathrm{idf}(t) = \log \frac{N + 1}{df(t) + 1} + 1
$$

$$
\mathrm{tfw}(t, d) = 1 + \log(tf(t, d))
$$

$$
\mathrm{score}(d, q) = \sum_{t \in q^{+} \cap d} \mathrm{tfw}(t, d) \cdot \mathrm{idf}(t)
$$

Где:

- $N$ — общее число документов в коллекции.
- $df(t)$ — число документов, в которых встречается терм $t$.
- $tf(t, d)$ — число вхождений термa $t$ в документ $d$.
- $q^{+}$ — множество положительных термов запроса, то есть термов, не находящихся под отрицанием `NOT`.

Практически это означает:

- если термин не встречается в документе, его вклад в score равен 0;
- если термин встречается 1 раз, вклад по tf равен 1;
- если термин встречается много раз, рост веса замедляется логарифмом;
- редкие термины получают больший вес за счёт idf.

Такая схема хорошо подходит как первый рабочий вариант ранжирования поверх булева поиска, потому что:
- не даёт деления на ноль;
- не обнуляет вклад редких терминов;
- легко вычисляется по обратному и координатному индексам.


## 🚀 HTTP API

```python
# POST /search
{ "query": "title::литва AND москва" }

# Ответ
{
  "query": "title::литва AND москва",
  "total_results": 2,
  "results": [
    {
      "id": "123",
      "title": "Литва — статья",
      "url": "https://...",
      "score": 0.0,
      "snippet": "Начало текста..."
    }
  ]
}
```

### Добавление новых документов

```python
# POST /api/documents

  "id": "new_doc_1",
  "content": "Content of the document...",
  "fields": {
    "title": "Document Title",
    "url": "https://..."
  }
}
```

## 🔧 Конфигурация и настройка

### Параметры поиска
- `max_distance`: максимальное расстояние между терминами (по умолчанию: без ограничений)
- TF-IDF параметры настраиваются в классе `DocumentIndexer`

### Расширение функциональности
Система спроектирована для легкого расширения:

1. **Поддержка опечаток**: добавить edit distance в `BooleanQueryParser`
2. **Джокерные символы**: расширить парсер запросов для `*` и `?`
3. **Фразовые запросы**: добавить поддержку кавычек `"exact phrase"`
4. **Автодополнение**: создать индекс префиксов терминов
5. **Синонимы**: интегрировать словарь синонимов
